# Roadmap

AI Refer is intentionally small at first. The goal is to build a useful public
reference base without turning it into a pile of prompts or source links.

## Current Foundation

- Meta guideline for creating packs.
- UI/UX fundamentals.
- Web app UI.
- Mobile app UI.
- UX design.
- Product requirements.
- Code architecture.
- Frontend architecture.

## Recommended Next Packs

### Component Design

Guidance for reusable components, variants, props, composition, accessibility,
states, tokens, documentation, examples, and visual QA.

### Code Review Checklist

Guidance for AI and humans reviewing changes for behavior, tests, architecture,
security, performance, migrations, observability, and maintainability.

### Desktop App UI

Guidance for Electron, Tauri, native-like desktop workflows, menus, shortcuts,
window behavior, dense tools, settings, local files, and offline-first states.

## Later Packs

- API design.
- Testing strategy.
- Security and privacy basics.
- AI app UX.
- Design systems.
- Data-heavy product UX.
- Documentation writing.
- Release and migration planning.

## Not Now

- Private prompt storage.
- User upload and parsing UI.
- Dynamic crawler service.
- Transcript archive.
- Large hosted database of raw references.

These can be revisited after the public pack format, validation, and contribution
workflow are stable.
